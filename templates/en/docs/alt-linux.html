Alt Linux Documentation

This section documentation contains information about Alt Linux.

There are so-called "subsections" in the section, which will be divided into elements of the settings of various packages, by the system itself with examples in the form of "pieces" of code and screenshots.

Alt Linux is a domestic operating system based on the Linux kernel and the Mandriva distribution.

The use of Alt at the moment is system administration to replace Windows Server.

Basic Configuration

The basic setup includes naming the system, creating users and assigning them as local administrators of the system other than the superuser. Users who will be considered local will be added to the sudoers file.

And so, let`s start with the simplest, the name of system:

hostnamectl set-hostname [full_name_with_domain]
# The hostname assignment is most often performed together with the domain.

Assigning a system name is necessary to identify it, for example, when a computer is in a domain, the main controller will read the names of the systems and identify them in a similar way.

Creating a user:

useradd [NAME] -m -U -s /bin/bash

The -m key is required to create the user's home directory, the -U key is required to automatically create a group for the created user

the -s key is required to specify the shell.

Adding the user to the "wheel" group:

usermod -aG wheel [NAME]

This command allows us to transfer the user to a special group of "administrators" (referred to in the system as the "wheel")

Granting admin privileges:

echo "[NAME] ALL=(ALL:ALL) NOPASSWD: ALL" >> /etc/sudoers

This command allows us to transfer the user to a special group of "administrators" (referred to in the system as the "wheel")

Network Configuration

Addressing in Alt is based on manually configuring devices and specifying IP addresses on the system.

For example, configure the interface with the IP address on machine:


mkdir /etc/net/ifaces/[NAME-INTERFACE] # Create a folder for each interface
cat < /etc/net/ifaces/[NAME-INTERFACE]/options # Making a configuration file for the first file
    > BOOTPROTO=static 
    > TYPE=eth
    > CONFIG_IPV4=yes
    > DISABLED=no
    > NM_CONTROLLED=no
EOF                    

or

vim /etc/net/ifaces/[NAME-INTERFACE]/options
> BOOTPROTO=static
> TYPE=eth
> CONFIG_IPV4=yes
> DISABLED=no
> NM_CONTROLLED=no

# ESC and write :wq! - exit VIM with save.

BOOTPROTO is responsible for configuring the adapter itself, in this case, the static address.

If you need a dynamically assigned DHCP, then write dhcpv4 / dhcp

TYPE - specifies the adapter type.

DISABLED - whether the adapter will be turned off or not

NM_CONTROLLED - whether the adapter will be controlled by the NetworkManager

CONFIG_IPV4 - enables IPv4 configuration

Assign an IP address:

echo "[IP]/[Mask]" > /etc/net/ifaces/[INTERFACE-NAME]/ipv4address

Assign a gateway:

echo "default via [IP-Gateway]" > /etc/net/ifaces/[INTERFACE-NAME]/ipv4route

Restart the service responsible for the network:

systemctl restart network

Check the status of the service:

systemctl status network

Check the assigned:

ip -c --br a # we are looking for the assigned IP address
ip -c r # we are looking for the assigned gateway

Temporary configuration for NAT addressing

Temporary setup is necessary for an emergency internet connection.

However, all the settings are fictitious and they work until the machine restarts. For security reasons, it is recommended to use this method rather than manual assignment using etcnet.

See, this is how it works:

ip link add link [NAME-INTERFACE] name [NAME-INTERFACE].[ID VLAN] type vlan id [ID VLAN] # make a link from the adapter to the additional
ip link set dev [NAME-INTERFACE] up # raise the main adapter
ip link set dev [NAME-INTERFACE].[VLAN ID] up # raise the reference adapter
ip addr add [IP address]/[Mask] dev [NAME-INTERFACE].[VLAN ID] # assign the IP address to the reference adapter
ip route add 0.0.0.0/0 via [Gateway IP address] # assigning a route

Open vSwitch - basic switching in Linux

Open vSwitch is most important package for networking in Linux, this is a virtual switch.

It is necessary to install the package:

apt-get install openvswitch

You need assign all interfaces to the virtual switch.

mkdir /etc/net/ifaces/[INTERFACE-NAME]
# Making a configuration file
cat < /etc/net/ifaces/[INTERFACE-NAME]/options
    TYPE=eth
    BOOTPROTO=static
EOF

cp /etc/net/ifaces/[First INTERFACE-NAME]/options /etc/net/ifaces/[Next INTERFACE-NAME]/options # Moving the configuration file to all other
systemctl restart network # restarting the main network service

For continue, you need create a management interface:

mkdir /etc/net/ifaces/MGMT
# Making a configuration file
vim /etc/net/ifaces/MGMT/options
> TYPE=ovsport
> BOOTPROTO=static
> CONFIG_IPV4=yes
> BRIDGE=NAME-BRIDGE OR SYSTEM 
> VID=VLAN ID

Where TYPE is, we specify the adapter type, in our case it is ovsport, because it will use the Open vSwitch.

Dont forget delete special option in /etc/net/ifaces/default/options:

vim /etc/net/ifaces/default/options
> OVS_REMOVE=no

We cut off the Open vSwitch service management, because we configure it ourselves.

Restart the management service:

systemctl restart network

Now you can see the formatted output of IP addresses on the system:

ip -c --br a

Now, we can create a special settings for Open vSwitch:

ovs-vsctl add-br [NAME-BRIDGE] # create a bridge
ovs-vsctl add-port [NAME-BRIDGE] [INTERFACE-NAME] # add an interface to the bridge
or 
ovs-vsctl add-port [NAME-BRIDGE] [INTERFACE-NAME] [TRUNK/TAG]=[NUMBER or NUMBERS with commas] # add an interface to the bridge with trunk or tag
# For example:
ovs-vsctl add-port SW ens19 trunk=3213,3212,3211

Here you create a port on the bridge with an indication of the interface being conducted and the type of port conduction, trunk - trunk.

When specifying a tag, we can specify a single VLAN ID

You can check settings Open vSwitch with command ovs-vsctl show

Dont forget enable mod in kernel 8021q:

modprobe 8021q
echo "8021q" | tee -a /etc/modules # fully enable it on an ongoing basis
lsmod | grep "8021q" # checking the module operation
Important! If the 8021q kernel module settings are not installed, the ping-pong packages will not pass through.

You can configure STP protocol in Open vSwitch with basicly settings and command:

ovs-vsctl set bridge [NAME-BRIDGE] stp_enable=true # setting the STP protocol to TRUE
ovs-vsctl set bridge [NAME-BRIDGE] other_config:stp-priority=[16384 OR ANY NUMBER] # specifying a number less than if it must be rooted
Important! STP settings must be performed on each virtual switch without exceptions, otherwise, in case of any errors, a switching loop will appear, from which you can exit using the "Troubleshooting" method, through ARPING and other methods.

DNS Server Configuration

Setting up a DNS server using BIND with fallback and security configuration.

Important! Incorrect configuration may break everything. Be careful with config files!

Initial setup and configuration

# Temporary DNS to install packages
echo "nameserver 77.88.8.8" > /etc/resolv.conf
            
apt-get update && apt-get install -y bind bind-utils
            
vim /etc/bind/options.conf
> listen-on { 127.0.0.1; [IP Machine]; };
> listen-on-v6 { any; };
> forwarders { 77.88.8.8; };
> allow-query { [First IP-Subnet]; [Second IP-Subnet]; };
> allow-transfer { [Secondary IP DNS Server]; };

:wq!

systemctl enable --now bind

cat < /etc/net/ifaces/[NAME-INT]/resolv.conf
    search [DOMAIN]
    nameserver [IP of local DNS server]
EOF

systemctl restart network bind

ping -c3 ya.ru

Backup DNS server (secondary)

vim /etc/bind/options.conf
> listen-on { 127.0.0.1; [IP Machine]; };
> listen-on-v6 { none; };
> forwarders { 77.88.8.8; };
> allow-query { [First IP-Subnet]; [Second IP-Subnet]; };
> allow-transfer { none; };

control bind-slave enabled

cat < /etc/net/ifaces/[NAME-INT]/resolv.conf
search [DOMAIN]
nameserver [Primary DNS]
nameserver [Secondary DNS]
EOF

systemctl restart network && systemctl enable --now bind
Bind error fix: If BIND throws permission errors, run: 
control bind-chroot disabled

NTP Protocol Configuration

Setting up an NTP server using Chrony to sync time with an external server.

Important! Ensure correct NTP server settings to maintain accurate time synchronization.

Initial Setup and Configuration

apt-get install chrony # installing a special package

vim /etc/chrony.conf  
> server [IP NTP Server] iburst prefer # specify as preferred server
> local stratum [N] # specify the stratum value if necessary
> allow [IP-Subnet] # specify the allowed subnets

:wq! - exit VIM with save

systemctl enable --now chronyd # enabling and adding chronyd to startup

chronyc sources # checking the functionality

timedatectl # checking the time zone

# On other machines that need to be connected to NTP
apt-get install chrony 
vim /etc/chrony.conf 
> server [IP NTP Server] iburst prefer 

:wq! 

systemctl enable --now chronyd # enabling and adding chronyd to startup
Fix for NTP Errors: If Chrony doesn't sync, check your configuration or try restarting the service.

Syncing with External NTP Server

# To sync with the NTP server, use the following commands:

chronyc tracking # show current synchronization status
chronyc sources -v # show NTP sources

systemctl restart chronyd # restart Chrony if there are sync issues

# Check synchronization status again
chronyc tracking

SAMBA AD Domain Infrastructure

Setting up the Samba domain infrastructure with DNS integration and Kerberos configuration.

Install Configuration Join Backup DC Shared Folder
Important! Ensure BIND is configured properly before setting up Samba for the domain to function correctly.

Samba AD Installation

apt-get install -y task-samba-dc # installing the SAMBA AD package
control bind-chroot disabled # disable CHROOT mode for BIND

grep -q KRB5RCACHETYPE /etc/sysconfig/bind || echo 'KRB5RCACHETYPE="none"' >> /etc/sysconfig/bind 
# Disabling krb5kdc cache type from BIND, allowing Samba to access Kerberos.

grep -q 'bind-dns' /etc/bind/named.conf || echo 'include "/var/lib/samba/bind-dns/named.conf";' >> /etc/bind/named.conf 
# Add Samba file paths to BIND config

vim /etc/bind/options.conf 
> tkey-gssapi-keytab "/var/lib/samba/bind-dns/dns.keytab"; 
> minimal-responses yes;

> logging { 
> category lame-servers { null; };
> }

:wq! # exit VIM with save

systemctl stop bind # stop BIND

rm -f /etc/samba/smb.conf 
rm -rf /var/lib/samba
rm -rf /var/cache/samba
mkdir -p /var/lib/samba/sysvol 
# Remove old config files, directories, and create SYSVOL folder

samba-tool domain provision # initialize the domain
Important! When choosing the module, because we used DNS in the early settings. DNS Backend: BIND9_DLZ.

We can continue the configuration.

Samba AD Configuration

    systemctl enable --now samba && systemctl start bind 
# Launching Samba AD with the addition of autorun and bind

cp /var/lib/samba/private/krb5.conf /etc/krb5.conf # copy the Kerberos configuration file from the Samba private directory to the main one.

# Adding records like A, PTR, and CNAME:
    
# Records of type A are added as follows:
samba-tool dns add 127.0.0.1 [domain-name] [machine-name] A [IP machine] -U[Name Admininstrator] -p[Password]
# Adding entry A. Example of filling in:
samba-tool dns add 127.0.0.1 gosgov.ru client.gosgov.ru A 38.0.101.76 -Uadmin -padmin

samba-tool dns query 127.0.0.1 [domain-name] @ A # checking for adding an entry of type A
    
# PTR type records are added as follows:
samba-tool dns zonecreate 127.0.0.1 [Reverse-IP].in-addr.arpa # Adding a recording zone 
    
samba-tool dns zonelist 127.0.0.1 # Checking if a zone has been added

samba-tool dns add 127.0.0.1 [Reverse-IP].in-addr.arpa [Last-Host-NumIP] PTR [Full-Domain-Name with Hostname] 
# Directly adding a PTR record. Example of filling in:
samba-tool dns add 127.0.0.1 10.10.10.in-addr.arpa 1 PTR client.gosgov.ru

# CNAME type records are added as follows:
samba-tool dns add 127.0.0.1 [Domain Name] [NAME-SITE] CNAME [Full-Domain-Name with Hostname] -U[Name Admininstrator] -p[Password]
# Adding a CNAME record. Example of filling in:
samba-tool dns add 127.0.0.1 gosgov.ru www CNAME server.gosgov.ru -Uadmin -padmin

# Create groups and users:
samba-tool group add [NAME] -U[Name Admininstrator] -p[Password] 
# Create a user group. Example of filling in:
samba-tool group add users -Uadmin -padmin

# Create Users:
samba-tool user add [NAME] [Password] -U[Name Admininstrator] -p[Password]
# Example of filling in:
samba-tool user add user resu -Uadmin -padmin

# Disabling EXPIRATION options and assigning a user to a group:
samba-tool user setexpiry [NAME-USER] --noexpiry 
# Change the EXPIRATION parameter. Example of filling in:
samba-tool user setexpiry user --noexpiry

samba-tool group addmembers "[NAME-GROUP]" [NAME-USER] 
# Adding a user to a group. Example of filling in:
samba-tool group addmembers "users" user

samba-tool ou add 'OU=[NAME]'
# Adding a division. Example of filling in:
samba-tool ou add 'OU=IT'
Fix for BIND or Samba Errors: If issues persist, ensure all configuration steps were followed correctly.

Backup Implementation

Setting up a backup system using systemd service and timer units for automated archiving.

Important! Ensure /opt/data contains critical files before proceeding.

Basic Backup Service

Creates a compressed archive of /opt/data on demand.

# Create backup directory
mkdir -p /var/bac/  

# Service unit configuration
vim /etc/systemd/system/backup.service

[Unit]
Description=Backup service for /opt/data

[Service]
Type=oneshot
ExecStart=/bin/tar -czf "/var/bac/SAMBA.tar.gz" /opt/data 

[Install]
WantedBy=multi-user.target

:wq!

systemctl daemon-reload
systemctl enable --now backup

Automated Daily Backups

Runs the backup daily at 20:00 with persistent logging.

# Timer unit configuration
vim /etc/systemd/system/backup.timer

[Unit]
Description=Daily backup at 20:00

[Timer]
OnCalendar=*-*-* 20:00:00
Persistent=true
Unit=backup.service

[Install]
WantedBy=timers.target

:wq!

systemctl daemon-reload
systemctl enable --now backup.timer
Pro Tip: Use journalctl -u backup.service to check backup logs.

Domain Management

Managing domain policies and shared resources using ADMC and GPUI tools.

This guide assumes you have administrative privileges in the domain.

Initial Setup

# Update and install required packages
apt-get update && apt-get install -y gpupdate admc gpui

# Enable group policy module
gpupdate-setup enable

# Get Kerberos ticket (run as admin)
kinit [AdminName]@[DOMAIN-NAME]

Group Policy Configuration

To manage policies through ADMC:

  1. Launch ADMC from applications
  2. Navigate to "Group Policy Objects" → desired section (e.g. IT)
  3. Right-click → "Create a policy and link it to a division"
  4. Name the policy and click OK
  5. Right-click policy → "Edit..." to launch GPUI

Example: Desktop Background Policy

Path in GPUI:

Computer → Administrative Templates → ALT System → Mate Settings → Desktop Background

  • Set status to "Enabled"
  • Fill "File" section
  • Check "Block" option

Network Restrictions

Path in GPUI:

ALT System → Polkit Rules → NetworkManager Restrictions

  • Enable "Limitation of network mode switching"
  • Set "Disable options" to No
  • Check "Block"

Shared Folder Configuration

# Create and link new policy
    # Right-click department → "Create a policy and link to this department"

In GPUI:

Computer → Settings → System Settings → Network drives

Required Settings:

  • Action: Update
  • Path: smb://server-name/samba
  • Reconnect: Enabled
  • Name: samba

Drive Options:

  • Disk Name: Use [Letter] (e.g. F)
  • Show disk
  • Show all disks
After configuration, restart computers to apply changes.
UI can be translated to Russian via: Settings → Language (ADMC) or View menu (GPUI)

Zabbix Monitoring System

Centralized monitoring with PostgreSQL backend.

Requires PostgreSQL server installed first.

Database Setup

# Install packages
apt-get update && apt-get install -y postgresql16-server zabbix-server-pgsql

# Initialize PostgreSQL
/etc/init.d/postgresql initdb
systemctl enable --now postgresql.service

# Create Zabbix user and DB
su - postgres -s /bin/sh -c 'createuser --no-superuser --no-createdb --no-createrole --encrypted -P zabbix'
su - postgres -s /bin/sh -c 'createdb -O zabbix zabbix'

# Import schemas
su - postgres -s /bin/sh -c 'psql -U zabbix -f /usr/share/doc/zabbix-*/schema.sql zabbix'
su - postgres -s /bin/sh -c 'psql -U zabbix -f /usr/share/doc/zabbix-*/images.sql zabbix'
su - postgres -s /bin/sh -c 'psql -U zabbix -f /usr/share/doc/zabbix-*/data.sql zabbix'

Web Interface

apt-get install -y apache2 apache2-mod_php8.2 # installing the package apache2 and addon apache2 mod_php
