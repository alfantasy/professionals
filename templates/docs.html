This section contains information about Alt Linux.

There are so-called "subsections" in the section, which will be divided into elements of the settings of various packages, by the system itself with examples in the form of "pieces" of code and screenshots.

Alt Linux is a domestic operating system based on the Linux kernel and the Mandriva distribution.

The use of Alt at the moment is system administration to replace Windows Server.

The basic setup includes naming the system, creating users, and assigning them as local administrators of the system other than the superuser. Users who will be considered local will be added to the sudoers file


And so, let's start with the simplest, the name of the system:


hostnamectl set-hostname [full_name.domain]
# The hostname assignment is most often performed together with the domain.

Assigning a system name is necessary to identify it, for example, when a computer is in a domain, the main controller will read the names of the systems and identify them in a similar way.

Creating a user:


useradd [NAME] -m -U -s /bin/bash 
# the -m key is required to create the user's home directory, the -U key is required to automatically create a group for the created user
# the -s key is required to specify the shell.

usermod -aG wheel [NAME] 
# This command allows us to transfer the user to a special group of "administrators" (referred to in the system as the "wheel")

echo "[NAME] ALL=(ALL:ALL) NOPASSWD: ALL" >> /etc/sudoers
# The command allows you to add a user to the sudoers file to access the system without entering a password. (sudo su -)

Addressing in Alt is based on manually configuring devices and specifying IP addresses on the system.



mkdir /etc/net/ifaces/[INTERFACE-NAME]                                    
vim /etc/net/ifaces/[INTERFACE-NAME]/options
> BOOTPROTO=static
> TYPE=eth
> DISABLED=no
> NM_CONTROLLED=no
> CONFIG_IPV4=yes

# BOOTPROTO configuration - is responsible for configuring the adapter itself, in this case, the static address.
# If you need a dynamically assigned DHCP, then write dhcpv4 / dhcp 
# TYPE - specifies the adapter type. 
# DISALBED - whether the adapter will be turned off or not
# NM_CONTROLLED - whether the adapter will be controlled by the NetworkManager
# CONFIG_IPV4 - enables IPv4 configuration 

:wq! - exit VIM with save. 

echo "[IP]/[Mask]" > /etc/net/ifaces/[INTERFACE-NAME]/ipv4address
echo "default via [IP-Gateway]" > /etc/net/ifaces/[INTERFACE-NAME]/ipv4route

systemctl restart network # restarting the service responsible for the network 
ip -c -br a # we are looking at the formatted output of the command indicating the adapters, status and its IP
ip -c r # we are looking at the routes

Temporary setup is necessary for emergency internet connection.


ip link add link [NAME-INTERFACE] name [NAME-INTERFACE].[ID VLAN] type vlan id [ID VLAN] # make a link from the adapter to the additional
ip link set dev [NAME-INTERFACE] up # raise the main adapter
ip link set dev [NAME-INTERFACE].[VLAN ID] up # raise the reference adapter
ip addr add [IP address]/[Mask] dev [NAME-INTERFACE].[VLAN ID] # assign the IP address to the reference adapter
ip route add 0.0.0.0/0 via [Gateway IP address] # assigning a route

Switching settings are needed for packet forwarding.


apt-get install openvswitch # install a special package

mkdir /etc/net/ifaces/[NAME-INTERFACE`S] # We create a folder for each interface                        
cat < /etc/net/ifaces/[NAME-INTERFACE]/options 
    TYPE=eth
    BOOTPROTO=static
EOF 
# Making a configuration file for the first file

cp /etc/net/ifaces/[First NAME-INTERFACE]/options /etc/net/ifaces/[Next NAME-INTERFACE]/options # Moving the configuration file to all other
systemctl restart network # restarting the main network service

mkdir /etc/net/ifaces/MGMT # creating the main management interface
vim /etc/net/ifaces/MGMT/options
> TYPE=ovsport
> BOOTPROTO=static
> CONFIG_IPV4=yes
> BRIDGE=NAME-BRIDGE OR SYSTEM 
> VID=VLAN ID  

# where TYPE is, we specify the adapter type, in our case it is ovsport, because it will use openvswitch
# BOOTPROTO - we already know, this is an indication of the settings for the adapter
# CONFIG_IPV4 - enabling IP address assignment 
# BRIDGE - enter the name of the bridge
# VID - specify the VLAN number 

:wq! - exit VIM with save 

vim /etc/net/ifaces/default/options
> ...
> OVS_REMOVE=no

# We cut off the OVS service management, because we configure it ourselves.

:wq! - exit VIM with save

systemctl restart network # restart the management service

ip -c -br -4 a # looking at the formatted output of IP addresses on
ip -c r interfaces # looking at routes
ovs-vsctl show # checking openvswitch settings

ovs-vsctl add port [NAME BRIDGE] [INTERFACE] [TRUNK/TAG]=[ID VLAN WITH ,] 
EXAMPLE: 
    ovs-vsctl add port SW ens19 trunk=3213,3214,3215
# Here you create a port on the bridge with an indication of the interface being conducted and the type of port conduction, trunk - trunk.
# When specifying a tag, we can specify a single VLAN ID 

modprobe 8021q # enabling the kernel module 
echo "8021q" | tee -a /etc/modules # fully enable it on an ongoing basis

ovs-vsctl show # viewing openvswitch
lsmod | grep "8021q" # checking the module operation

ovs-vsctl set bridge [NAME BRIDGE] stp_enable=true # setting the STP protocol to TRUE
ovs-vsctl set bridge [NAME-BRIDGE] other_config:stp-priority=[16384 OR ANY NUMBER] # specifying a number less than if it must be rooted

This section contains information about configuring the DNS server.

Important! When changing the configuration, be careful with the configuration files, otherwise everything will break and nothing will work. :(


# Temporary configuration specifying the DNS server from Yandex for installing packages 
echo "nameserver 77.88.8.8" > /etc/resolv.conf 

apt-get update && apt-get install -y bind bind-utils # installing BIND packages and special utilities from the same

# Open the main Bind configuration file and change certain settings:
vim /etc/bind/options.conf 
> listen-on { 127.0.0.1; [IP Machine]; };
> listen-on-v6 { any; }; # but you can also specify none, nothing will change :D 

> forwarders { 77.88.8.8; }; # if we do not specify a redirector here, there will be no Internet and no standard zones

> allow-query { [First IP-Subnet]; [Second IP-Subnet]; }; # specify the IP addresses that will have access to the DNS server
> allow-transfer { [Secondary IP DNS Server from local machine]; }; # specify the backup DNS server, if available. 

:wq! - exit VIM with save

systemctl enable --now bind # enable and add bind to startup

cat < /etc/net/ifaces/[NAME-INT]/resolv.conf 
    search [DOMAIN]
nameserver [IP DNS Server from local machine]
EOF
# where [IP DNS Server from local machine] - we specify the IP of our machine on which we configured the whole thing. 

systemctl restart network bind # restart the necessary services 

# In order to check, you can ping the same Yandex 
ping -c3 ya.ru

# At the exit, you should receive successful pings on the DNS of the site.

There are no distinguishing features when setting up a backup DNS, but we will provide the settings briefly.



vim /etc/bind/options.conf 
> listen-on { 127.0.0.1; [IP Machine]; };
> listen-on-v6 { none; }; 

> forwarders { 77.88.8.8; }; 

> allow-query { [First IP-Subnet]; [Second IP-Subnet]; };
> allow-transfer { none; }; 
# Is none here, because it should not refer to the first DNS, because it is a backup, nothing needs to be forwarded.

control bind-slave enabled # enable the SLAVE mode and set the control options for it 

cat < /etc/net/ifaces/[NAME-INT]/resolv.conf
    search [DOMAIN] 
    nameserver [First DNS Server from local machine]
    nameserver [Second DNS Server from local machine]
EOF
systemctl restart network && systemctl enable --now bind # restart necessary services

Important! In case of a bind malfunction, namely an error with access rights, you must enter the following:

control bind-chroot disabled

We fully devote our time to a separate section, which will contain settings designed for the Samba domain infrastructure.

The performance of Samba depends entirely on the configured BIND, and if suddenly it doesn't work, then the domain won't work either.

If BIND swears at /etc/bind/named.conf files, then do the following:


    control bind-chroot disabled
    # # Setting this setting completely disables BIND's sudden uncontrolled behavior.
    # If it is not disabled, BIND uses the "CHROOT" mode, although it has its own user.

Перейдем к самой настройке Samba AD



    apt-get install -y task-samba-dc # installing the SAMBA AD package
    control bind-chroot disabled # turn off the CHROOT mode (mentioned above)
    
    grep -q KRB5RCACHETYPE /etc/sysconfig/bind || echo 'KRB5RCACHETYPE="none"' >> /etc/sysconfig/bind 
    # Disabling the krb5kdc cache type from BIND, thereby providing Samba access to the Kerberos protocol.
    
    grep -q 'bind-dns' /etc/bind/named.conf || echo 'include "/var/lib/samba/bind-dns/named.conf";' >> /etc/bind/named.conf 
    # We enter file names from Samba to the main BIND file.
    
    # Edit the file responsible for BIND:
    vim /etc/bind/options.conf 
    > tkey-gssapi-keytab "/var/lib/samba/bind-dns/dns.keytab"; 
    > minimal-responses yes; 
    
    > logging { 
    > category lame-servers { null; };
    > }
    
    :wq! # выход из VIM с сохранением
    
    # Disable logging, enter the KEY KEY and enable minimal responses.
    
    systemctl stop bind # stopping BIND
    
    rm -f /etc/samba/smb.conf 
    rm -rf /var/lib/samba
    rm -rf /var/cache/samba
    mkdir -p /var/lib/samba/sysvol 
    # Delete the old configuration files, private directories, and create the SYSVOL folder.
    
    samba-tool domain provision # initializing the domain. 
    # IMPORTANT! When choosing the module, because we used DNS in the early settings:
    # DNS backend: BIND9_DLZ 
    
    systemctl enable --now samba && systemctl start bind 
    # Launching Samba AD with the addition of autorun and bind
    
    cp /var/lib/samba/private/krb5.conf /etc/krb5.conf # copy the Kerberos configuration file from the Samba private directory to the main one.
    
    # Adding records like A, PTR, and CNAME:
    
    # Records of type A are added as follows:
    samba-tool dns add 127.0.0.1 [domain-name] [machine-name] A [IP machine] -U[Name Admininstrator] -p[Password]
    # Adding entry A. Example of filling in:
    samba-tool dns add 127.0.0.1 gosgov.ru client.gosgov.ru A 38.0.101.76 -Uadmin -padmin
    
    samba-tool dns query 127.0.0.1 [domain-name] @ A # checking for adding an entry of type A
    
    # PTR type records are added as follows:
    samba-tool dns zonecreate 127.0.0.1 [Reverse-IP].in-addr.arpa # Adding a recording zone 
        
    samba-tool dns zonelist 127.0.0.1 # Checking if a zone has been added
    
    samba-tool dns add 127.0.0.1 [Reverse-IP].in-addr.arpa [Last-Host-NumIP] PTR [Full-Domain-Name with Hostname] 
    # Directly adding a PTR record. Example of filling in:
    samba-tool dns add 127.0.0.1 10.10.10.in-addr.arpa 1 PTR client.gosgov.ru
    
    # CNAME type records are added as follows:
    samba-tool dns add 127.0.0.1 [Domain Name] [NAME-SITE] CNAME [Full-Domain-Name with Hostname] -U[Name Admininstrator] -p[Password]
    # Adding a CNAME record. Example of filling in:
    samba-tool dns add 127.0.0.1 gosgov.ru www CNAME server.gosgov.ru -Uadmin -padmin
    
    # Create groups and users:
    samba-tool group add [NAME] -U[Name Admininstrator] -p[Password] 
    # Create a user group. Example of filling in:
    samba-tool group add users -Uadmin -padmin
    
    # Create Users:
    samba-tool user add [NAME] [Password] -U[Name Admininstrator] -p[Password]
    # Example of filling in:
    samba-tool user add user resu -Uadmin -padmin
    
    # Disabling EXPIRATION options and assigning a user to a group:
    samba-tool user setexpiry [NAME-USER] --noexpiry 
    # Change the EXPIRATION parameter. Example of filling in:
    samba-tool user setexpiry user --noexpiry
    
    samba-tool group addmembers "[NAME-GROUP]" [NAME-USER] 
    # Adding a user to a group. Example of filling in:
    samba-tool group addmembers "users" user
    
    samba-tool ou add 'OU=[NAME]'
    # Adding a division. Example of filling in:
    samba-tool ou add 'OU=IT'

Tunneling involves connecting one router to another by redirecting packets through an ISP.

As a rule, packets do not linger at the provider, but immediately find the recipient, bypassing the provider - therefore, the packet will not be visible on the provider's side.


conf t # switching to
interface tunnel.0 # switching to
desc "[Description/Purpose]" # purpose
ip address add [IP]/[MASK] # assign an IP address
ip mtu [MTU] # assign the MTU
ip tunnel [IP from Provider] [IP Second Router from Provider] mode gre # Assigning a tunnel between the routers
end # complete the full configuration and exit configuration mode
write # configuration entry
# where the IP addresses are specified, the IP addresses of the routers facing the provider are indicated there
